# 供电设备图像异动检测

本项目比较供电设备的标准图和实时图，目标是在**先可靠对齐图片**的前提下，检测指定部件是否发生可见变化。

项目处于重构阶段。旧的 C/C++ 原型仅作为诊断和参考；后续新模块统一采用 Python 实现。

## 核心原则

1. 调用方提供标准图、实时图和标准图中的部件框；当前项目不依赖设备编号、机位编号或外部身份库。
2. 程序先判断两张图能否可靠重合，再比较部件区域。
3. 无法可靠比较时必须明确返回“检测不可用”，不得显示为“无异动”。
4. 每次有效比较都保存对齐图、有效区域图、差异区域图、差异热力图和标注图，便于追溯。
5. 内部图片和由其生成的报告只允许在授权的本地或内网环境中处理，不能上传或外发。

## 单部件模块

代码位于 [`imagecmp_py/`](imagecmp_py/)。其输入为：

- 标准图路径；
- 实时图路径；
- 标准图中的一个部件框，格式为 `类别 中心横坐标 中心纵坐标 宽 高`，坐标已归一化；
- 带版本号的配置文件；
- 本地证据输出目录。

其输出为下列三种状态之一：

| 状态 | 含义 |
| --- | --- |
| `no_change_high_confidence` | 图片可靠对齐、部件区域可比较，且未发现达到当前配置门槛的差异。 |
| `change_detected` | 图片可靠对齐，并发现一个或多个差异区域。 |
| `detection_unavailable` | 图片可读但无法可靠对齐或可比较区域不足；不得显示为正常。 |

目前随代码提供的开发配置仅用于跑通流程和合成测试，尚未经过内部正常样本标定，不能作为现场生产配置，也不能据此宣称准确率、召回率或漏检率。

## 运行

需要 Python 3.12 或兼容版本。

```powershell
python -m pip install -r .\imagecmp_py\requirements.txt
python .\imagecmp_py\test_service.py
```

使用一组图片进行单部件比较：

```powershell
python .\imagecmp_py\cli.py compare `
  --standard .\test\temp.jpg `
  --live .\test\live.jpg `
  --roi "17 0.5 0.5 0.4 0.5" `
  --config .\imagecmp_py\configs\development-default-v1.json `
  --output .\output\manual-check
```

详细输入、输出和证据图说明见 [`imagecmp_py/README.md`](imagecmp_py/README.md)。

## 项目路线

1. 建立单部件比较基础。
2. 增加标定和日常检测模式，以及整图多部件汇总。
3. 建立可靠对齐链路，并使用内部正常样本标定对齐质量门槛。
4. 在可靠对齐后逐步增加颜色、结构、边缘和深层图像特征等差异证据。
5. 完成本地回归、不可检测率评估和上线可用性判断。

现有模型文件可用于图像几何匹配、边缘或通用图像特征等辅助能力；它们不是设备身份识别模型，也不是通用缺陷分类模型。
