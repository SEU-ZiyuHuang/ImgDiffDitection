## 四、四个方案的详细介绍

### 4.1 共性与差异

四个方案只修改**门禁阈值**，不改代码逻辑。每道门禁都是"指标低于阈值就拦下"（min-metric）或"指标高于阈值就拦下"（max-metric）。阈值推导基于 `component_observations.csv` 中剔除 2 例鸟害后的 769 个部件。

### 4.2 方案A：严格分位数（p05 / p95）

**设计逻辑**：对"越高越好"的指标（NCC、覆盖度、内点率等），取正常样本分布的第5百分位（p05）作为门槛——即 5% 的正常样本会被拦下；对"越低越好"的指标（重投影误差），取第95百分位（p95）。

**致命问题暴露**：这种做法的前提是"所有样本都是正常样本"，但 NCC 的原始分布被大量真实错配对污染。污染把左尾拖到了负数，导致算出的 `ncc_min = -0.18`，门禁形同虚设——之前实验证明的 81% 真实错配会被全部放进比较环节。同时 `resolution_scale_min = 0.92` 又严到荒谬（要求实时图分辨率不能比标准图低 8%）。

### 4.3 方案B：宽松分位数（p01 / p99）

**设计逻辑**：用更宽松的分位点（1%/99%），只拦极端异常值。

**致命问题**：`ncc_min = -0.36`，门禁完全失效，91% 的部件直接放行。所有被拦的错配对涌入比较环节，差异检测会产出巨量假异动框（错配的两张图在像素上完全不同），从而系统性制造假告警。

### 4.4 方案C：分案例类型

**设计逻辑**：在方案A的 p05/p95 基础上，按案例类型分组（日常/测温/表计/位置…），每类单独标定，样本不足30的组回落到全局阈值。

**问题**：多数组的样本量不够独立标定（测温只有20个部件），实质回落到全局；且继承方案A同样的"污染分布"问题。`ncc_min` 除 unspecified 组为 -0.30 外，其余全为 -0.18。

### 4.5 方案D（推荐）：语义+干净子集混合标定

**设计逻辑**：不盲从分位数公式，而是区分两类门槛：
- **能用分位数的**（重投影误差、内点率、有效重叠等配准几何指标）：取 p01/p99，并参考指标的业务语义做调整；
- **分位数失效的**（NCC/SSIM、空间覆盖度、有效分辨率）：要么用"干净子集"（独立证据确认配得上的 145 个部件）的分位点，要么用文档化的语义值。

**干净子集条件**：ECC 收敛 + ECC 相关性≥0.8 + 内点覆盖度≥0.2 + 内点率≥0.8。这 145 个部件可以高度确信是"真配上"的。

**干净子集上 NCC 的分布**：p01 = 0.097, p05 = **0.451**, 中位数 = 0.904, p95 = 0.983。而被拦的 315 个部件全部 <0.35，且没有一个能到 0.45。**两个人群在 0.35~0.45 之间天然分开**，说明当前的 ncc_min=0.35 已经坐在正确的位置，不需要动。

**方案D各门槛及依据**：

| 门禁 | 现值（占位） | 方案D | 依据 |
|---|---|---|---|
| reprojection_error_max | 3.0 | 1.5 | 全样本 p99 = 1.43 |
| spatial_coverage_min | 0.02 | 0.05 | 语义下限（分布 p05 = 0.0099 失效）；真正的防护靠父子图签名 + NCC |
| valid_overlap_ratio_min | 0.6 | 0.7 | 全样本 p01 = 0.73 |
| candidate_in_frame_ratio_min | 0.6 | 0.7 | p01 = 0.72 |
| roi_valid_overlap_ratio_min | 0.6 | 0.85 | p01 = 0.88 |
| ecc_correlation_min_when_converged | 0.2 | 0.15 | p05 = 0.14 |
| inlier_rate_min | 0.4 | 0.4（不变） | p05 = 0.44 / p01 = 0.19，现值合理 |
| effective_resolution_scale_min | 0.25 | 0.4 | 语义下限（0.4倍采样以下，螺栓/裂痕无法分辨） |
| effective_component_min_pixels | 24 | 48 | 语义下限；与数据 p01（46/51）一致 |
| **appearance_ncc_min** | 0.35 | **0.35（不变）** | 干净子集谷地验证（0.35位于两群空档，仅误拦~2-3%真正常，不漏真错配） |
| **appearance_ssim_min** | 0.2 | **0.2（不变）** | 干净子集 p01 = 0.229，现值≈p01 |

## 六、全量救援审计结果（全315个NCC被拦部件）

**方法**：对每个被 NCC 门禁拦下的部件，重跑配准后，用三种抗光照算法（直方图匹配、局部z-score归一化、梯度幅值NCC）重新计算相似度。只要有一种恢复 ≥0.35，判为"结构对得上、只是光照变了"（= 按业务规则不算异动）。

**精确结果**（全量，非抽样）：

| NCC桶 | 数量 | 救回 | 救回率 |
|---|---|---|---|
| <0（完全负相关） | 110 | 12 | 10.9% |
| 0 ~ 0.15 | 77 | 3 | 3.9% |
| 0.15 ~ 0.25 | 70 | 17 | 24.3% |
| 0.25 ~ 0.35 | 58 | 23 | 39.7% |
| **总计** | **315** | **55** | **17.5%** |

**含义**：
- **260个（82.5%）**: 换什么算法都对不上 → 真错配或真场景变化 → 报"异动-图像不匹配"正确
- **55个（17.5%）**: 结构一致、只是光照变化（日/夜差异） → 按业务规则不应报异动 → 需 #8 的光照归一化后回收到正常比较流程

**修正后的方案D"异动-图像不匹配"构成**：

| 成分 | 数量 |
|---|---|
| 真错配（抗光照验证失败） | 260 |
| 光照误伤（抗光照可恢复） | 55 |
| SSIM太低被拦（待查） | 8 |
| 部件大半出框 | 7 |
| **合计** | **330**（注：其中55个需#8修正） |

**全量结果CSV**：`D:\异动检测批量输出\ncc_rescue_full_20260722.csv`
