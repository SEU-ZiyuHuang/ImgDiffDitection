# Issue 8 总结：可信比较后的差异检测与可选模型对齐回退

> 完成日期：2026-07-23
> 对应 GitHub Issue：[Issue #8](https://github.com/SEU-ZiyuHuang/ImgDiffDitection/issues/8)
> 本文只记录本地受控数据上的运行行为和证据，不把候选数量说成异动数量，也不声称准确率、误报率、召回率或漏检率。

## 1. 这次解决了什么

Issue 7 已解决“这两张图能不能放心比较”：全图对齐、部件位置、有效像素和有效分辨率只要有一项证据不足，就输出“检测不可用”。Issue 8 在这个前提上处理另一个问题：即使图已经对齐，日照、曝光、阴影、局部反光、纹理抖动和拍糊，仍可能产生很多看起来像差异的像素。

本次代码实现了四个互相独立、按顺序工作的板块：

1. **可选的 SuperPoint + LightGlue 对齐回退。** 只有 ORB 对齐为 `unreliable`（不可靠）或 `unavailable`（不可用）时才试一次模型匹配。模型只提供全图对应点，仍必须通过 RANSAC（随机抽样一致算法）、投影几何、有效重叠和 ECC（增强相关系数）微调等原门槛，绝不直接放行。
2. **光照处理。** 在部件有效区域内将实时图亮度分布拉向标准图；光照差很大时，自动降低颜色证据的权重。
3. **局部结构比较。** 去掉缓慢变化的阴影后，比较边缘方向，避免把同一条线仅因曝光改变而误判为“线消失”。
4. **清晰度和候选分级。** 实拍部件明显比标准图模糊时，日常检测输出“检测不可用”；所有候选都保留，但只有面积和综合分数都达标的候选才可以触发业务结论。

## 3. 关键计算规则

### 3.1 先证明能比较，再比较差异

完整顺序是：选择参考图 → 全图对齐可信 → 部件映射可信 → 有效像素和有效分辨率可信 → 清晰度足够 → 计算差异候选。

因此，SuperPoint + LightGlue 找到匹配点也不代表可比较。它得到的点还要估计单应矩阵，并重新走所有全图门禁。模型回退失败时，保留原来的 ORB 诊断结果，不会以模型分数绕过门禁。

### 3.2 光照、颜色和结构

颜色通道在亮度分布拉齐后计算 Lab 色差。设标准图和处理后的实时图的 Lab 色差为 \(\Delta E\)，颜色门槛为 \(T_c\)，颜色证据为：

\[
C=\min(\Delta E/T_c,1)\times w_c
\]

其中 \(w_c\) 是颜色权重。两图整体亮度相差越大，\(w_c\) 越低，最低保留为 0.20，避免纯光照变化单独主导结论。

结构通道先减去局部模糊背景，再比较梯度方向。设两张图在某像素的梯度夹角余弦为 \(\cos\theta\)，边缘强度归一化为 \(e\)，结构证据为：

\[
S=\frac{1-\cos\theta}{2}\times e
\]

最终像素差异热力值是 \(H=\max(C,S)\)。只有 \(H\) 过差异门槛的像素形成候选区域；候选区域仍需经过面积和综合分数检查。

### 3.3 清晰度和候选分级

用拉普拉斯响应均值表示清晰度。标准图清晰度为 \(q_s\)，实时图清晰度为 \(q_l\)，清晰度比为：

\[
r=\frac{\min(q_s,q_l)}{\max(q_s,q_l)}
\]

本配置中 \(r<0.30\) 表示两图清晰度差得太多：日常检测输出 `detection_unavailable`，不报异动也不报无异动。

每个候选都写入 JSON、CSV 和标注图。只有“面积不少于 100 像素且综合分数不少于 0.30”的候选才标为 `decision_eligible=true`。这不是异动真值概率，只是当前版本配置下允许触发业务结论的证据门槛。

### 3.4 NCC 的含义

NCC（归一化互相关）用于部件映射时的外观一致性检查：

\[
\operatorname{NCC}(A,B)=
\frac{\sum_i(A_i-\bar A)(B_i-\bar B)}
{\sqrt{\sum_i(A_i-\bar A)^2}\sqrt{\sum_i(B_i-\bar B)^2}}
\]

它越接近 1，说明两块图像明暗变化趋势越相似；但单独一次 NCC 低不能直接证明异动。代码还会用亮度拉齐、局部对比和边缘复核，避免把光照变化叫作图像不匹配。

## 4. 四个历史阈值方案（Issue 7）及其边界

这四个方案是 Issue 7 用来选择“可信对齐与部件映射门槛”的 A/B/C/D 方案，不是四套 Issue 8 差异算法。它们的离线推演没有完整执行当时全部全图门禁，所以只能用于解释阈值选择，**不能把其中的异动、无异动或不可用数量当成正式检测结论**。

| 方案 | 做法 | 离线可用率 | 结论 |
| --- | --- | ---: | --- |
| A：严格分位数 | 对高分指标取第 5 百分位、对误差取第 95 百分位。 | 73.2% | 样本中混入错配图后，NCC 门槛被拉成负数，实际会放过大量不可信比较。 |
| B：宽松分位数 | 使用第 1 / 第 99 百分位，只拦极端值。 | 91.0% | 更容易放行错配图，安全性最差。 |
| C：按案例类型分组 | 日常、测温、表计等分别取分位数；样本不足时回退全局值。 | 72.7% | 多数分组样本不足，且继承 A 的污染问题。 |
| D：业务语义 + 干净子集 | 几何指标结合分位数；NCC、分辨率等使用高可信子集和业务下限。 | 52.4%（仅离线推演） | 被选为正式配置思路；关键门槛有可解释依据。 |

方案 D 的代表性门槛包括：重投影误差不大于 1.5 像素、空间覆盖不小于 0.05、全图有效重叠不小于 0.70、部件有效重叠不小于 0.85、有效分辨率比例不小于 0.40、部件最小尺寸为 48×48 像素、NCC 不小于 0.35、SSIM（结构相似性）不小于 0.20。它们现在都在版本化配置中，而不是散落在代码里。

正式的 Issue 7 D 配置重跑结果是：385 个请求完成 383 个，得到 771 个部件观察；580 个部件所在的全图对齐可信，378 个部件通过映射进入差异比较，产生 41,029 个原始候选。这个正式批次才是 Issue 8 的对比基线。

## 6. 无标签时，哪些结论可以说、哪些不能说

目前没有逐案例“真实异动 / 真实正常”的标签。因此可以报告：对齐可用率、模型救回数、门禁失败原因、候选数量、清晰度不足数量、运行时间和证据文件完整性。

不能报告：准确率、误报率、漏检率、精确率、召回率，或“318 个部件就是异动”。这些都需要独立人工确认的真值。

下一轮应建立小而可信的人工复核集，而不是先把 383 个案例全部凭感觉贴标签：

1. 按 ORB 可用、ORB 不可靠、模型救回、模糊、强光、不同设备类别分层抽样；
2. 两名熟悉设备的人员独立标记“同一部件且可比较 / 视角或部件不对应 / 无异动 / 有异动”，必要时标出异动区域；
3. 对分歧样本集中复核，形成冻结的验证集；
4. 用另一部分已确认样本调门槛，不能拿验证集反复调参；
5. 配置冻结后，才在未参与调参的验证集上计算准确率和召回率。

## 7. 下一步建议

1. **先决定模型回退是否进入日常路径。** 当前收益是 3/383 个案例，且 CPU 成本较高；在许可证确认前保持关闭。若要开启，先完成许可证审查，再给出可接受的单案例耗时上限。
2. **建立人工真值集。** 这是下一步最关键工作；没有它就不能判断 3,013 个可触发候选里有多少是真问题。
3. **复核 3 个模型救回案例。** 人工确认它们是否真的同一机位、同一设备、同一部件；这决定模型回退是否值得保留。
4. **用确认正常样本重新标定差异阈值。** 优先检查 318 个存在可触发候选的部件，区分光照残留、纹理噪声和真实变化。
5. **再进入 Issue 9。** 只有当 Issue 8 的阈值有真值支撑后，再评估 DINOv2、SAM 或其他更强模型是否能真正降低错误，而不是只让候选数量变化。
